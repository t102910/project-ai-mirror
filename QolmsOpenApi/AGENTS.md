# AI エージェント作業規則

このファイルは、このリポジトリで作業する AI エージェントが従うべき最上位ルールを定義します。

## 作業前の確認事項

実装に着手する前に、以下を必ず実施してください。

1. **関連ファイルの確認**
   - 対象のコントローラー・ビュー・モデルを事前に読み込む
   - 同名・類似名のファイルが既に存在しないか確認する

2. **影響範囲の確認**
   - 変更が複数ファイルにまたがる場合、先に影響ファイルの一覧を示す
   - DB・API・共有モデルへの影響がある場合は、着手前に整理して提示する

3. **ビルド・テスト観点の確認**
   - 型整合性・参照名の整合を事前に確認する
   - 既存テストへの影響がある場合は着手前に報告する

## 変更方針

- **最小変更原則** : 要求された箇所のみを変更する。依頼外の改善・リファクタリング・コメント追加は行わない
- **既存規約の優先** : 命名・フォーマット・設計パターンは既存コードに合わせる
- **推測での断定禁止** : 仕様が不明な場合は「要確認」と明示する
- **広範囲変更の事前整理** : 変更が広範囲になる場合は、影響範囲と方針を先に提示し、着手は確認後にする

## 命名規則の尊重

- 既存の命名規則・プレフィックス・サフィックスを事前に確認し、それに合わせる
- 確認なしにクラス名・メソッド名・ファイル名をリネームしない
- 新規命名時は業界標準として適切かを検討してから決定する

## 禁止事項

- 依頼されていない機能追加・設計変更
- 理由のないコメント付与・ドキュメント生成
- 一括置換・大量リネームの明示指示なしの実行
- 破壊的操作（ファイル削除・DB DROP 等）を確認なしに実行すること

### TFS 更新の取り込み手順

TFS 側で変更があったとき、タイムスタンプ基準でローカルへ上書き取り込みを行う。

```powershell
robocopy "C:\MGF\QOLMS\V1F472\Web\QolmsOpenApi" "C:\private-git\project-ai-mirror\QolmsOpenApi" /E /XD .git
```

取り込み後は差分を確認してからコミットする：

```powershell
cd "C:\private-git\project-ai-mirror"
git diff --stat QolmsOpenApi/
git add QolmsOpenApi/
git commit -m "chore: TFS 最新版を取り込み (YYYY-MM-DD)"
git push
```

### Git 運用方針

- `git commit` 後は `git push` まで自動実施する（確認不要）
- コミットメッセージは日本語で、変更内容を端的に書く
- `.gitignore` / `.gitattributes` / `AGENTS.md` / `.github/` はローカル管理ファイルのため、robocopy で上書きされない（TFS 側に存在しない）

- `JotoApp` 系ファイル（`JotoAppController.cs` 等）は LF 改行が混在している。編集時はエディタの改行設定に注意
- `packages/` は Git 管理対象外（NuGet パッケージは TFS 側の `packages/` フォルダを参照）
